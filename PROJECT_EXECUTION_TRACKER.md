## 📝 使用说明

### 执行方式

1. **查看当前完成度**
   ```bash
   cat PROJECT_EXECUTION_TRACKER.md | grep "当前完成度"
   ```

2. **检查待办任务**
   ```bash
   cat PROJECT_EXECUTION_TRACKER.md | grep "\[ \]"
   ```

3. **标记任务完成**
   - 将 `[ ]` 改为 `[✓]`
   - 更新对应的完成度

4. **查看完成进度**
   - 实时查看各阶段完成度
   - 根据完成度分配后续任务

### 任务状态说明

- `[ ]` - 待开始的任务
- `[✓]` - 已完成的任务
- `[⏸]` - 进行中的任务
- `[⚠️]` - 阻塞的任务
- `[❌]` - 失败的任务

## 🎯 阶段切换说明

系统共分 **10 个主要阶段**，每个阶段约 **10%** 的完成度：

### 自动检测完成度

当前系统会自动检测每个模块的完成度：
- 数据库模型数量
- API 接口数量
- 页面组件数量
- 代码文件行数

当某阶段的所有任务完成时，会自动标记该阶段完成，并切换到下一阶段。
